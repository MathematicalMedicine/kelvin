Running graphApp



There are two ways to run graphApp:

1. Run from the Cluster - Log onto the cluster with an X11 session and type 'graphApp' from any directory.   This will cause the program to be run from the cluster.  It will use Linux icons and menus.  The program speed and responsiveness may be a little slow when running from the cluster.

When using graphApp from the cluster, you may notice that the mouse turn yellow, or just completely disappears.  This problem is from a bug in Mac OSX.  To fix this, log into the cluster and type:

mkdir ~/.icons
cd ~/.icons
cp -R /usr/share/icons/HighContrast .
ln -s HighContrast default


2. Run from local machine - Copy the source files and run graphApp from a local computer.  The source files are located on the cluster at /usr/local/share/graphApp.  Source files are also located on the file server, under the software folder.  You will also need the following prerequisites in order to run graphApp from your computer.

    - Python 2.5
    - wxPython
    - Matplotlib 0.90.1
    - NumPy
    - Python Imaging Library (PIL)
    - PyXML    

If you are trying to install these packages on Mac OSX, all of them can be found here, except for PyXML. You will have to go to their homepage, here to download.  Once downloaded, follow their instructions to install the module.

Once the source files are copied and the prerequisites are installed, simply open a command prompt, goto the directory where the sources files are located, and type

python graphApp.py

Toolbar buttons


Tip: If you hover over a toolbar button for awhile, a tooltip of what it does appears.


   Open grx file
Opens a .grx file. A .grx file is a special file format used by graphApp to save the state of a graph, which can later be opened by graphApp.


   Import data from text file
Opens a text file containing data (See 'Import Data from File').


   Save grx file
Saves the currently shown graph as a .grx file.


   Export to image
Saves the currently shown graph as an image file.  Note that if no extension is given, the default image format is png.  Other formats are not recommended.


   Print
Print the current graph.


   Insert horizontal line
Use this button to insert a black horizontal line that runs through the entire graph.  When this button is pressed, a dialog box will appear asking what y-values to place the line(s).  If multiple horizontal lines are wanted, separate values with commas (e.g. '0.1, 0.2, 0.3').


   Zoom to Chromosome
These two widgets allow you to zoom in on one chromosome.  Use the drop-down box on the left to select which chromosome to zoom in on, then press the magnifying glass icon on the right to force the graph to only show that chromosome.


   Home
Clicking on this button will show the base view.  This usually means the x-axis shows all chromosomes in the graph, and the y-axis is set to 0 and 1.


   Back
Go to the previous view.  A new view is set when a zoom or a pan occurs.


   Forward
Go to the next view.  The opposite effect of the Back button.


   Pan
Press once to set the graph to pan mode.  Press again to get out of pan mode.  While in pan mode, you can drag the graph using the left mouse button.  There are two special options pertaining to this.  If you hold the 'x' button on the keyboard while dragging with the left mouse button, the graph will only move in the x direction.  If you hold the 'y' button on the keyboard while dragging, the graph will only move in the y direction.  Also in pan mode you can drag with the right mouse button.  Dragging with the right mouse button will case the limits of the axis to change.  For example, if you hold the right mouse button and drag to the right, the limits of the x-axis will change to smaller values, effectively 'zooming in'.


   Zoom Rectangle
Press once to set the graph to zoom mode.  Press again to get out of zoom mode.  While in zoom mode, you can left click and drag, and a selection box will appear.  When you release the left mouse button, the graph will zoom into the box that was selected.  Zooming with the right mouse button is also possible, but the function of it is not completely understood.  Usage of this feature is not recommended.


   Zoom X-Axis
Zoom in by only changing the limits of the x-axis.  Similar to Zoom Rectangle except that the height of the selection box will always be equal to the height of the graph.  Good for when you want to zoom without changing the limits of the y-axis.


   Zoom Y-Axis
Zoom in by only changing the limits of the y-axis.  Similar to Zoom Rectangle except that the width of the selection box will always be equal to the width of the graph.  Good for when you want to zoom without changing the limits of the x-axis.



   Annotate Point
Once this button is pressed, clicking on any point in the graph will result in that point being annotated.


   Annotate Threshold
Use this to annotate either all peaks above a certain threshold value, or all valleys below a certain threshold value.  When this button is clicked, and dialog box is displayed asking for the threshold value, and whether to annotate peaks or valleys.


   Annotate Peak
Annotate the highest peak in an area.  When this button is clicked, you can drag and create a selection box.  When the mouse button is released, the highest peak in the selection box is annotated.  The definition of a peak is a point where it is higher than both its neighbors.  If a point is at one of the ends of a chromosome, it only needs to be higher than its only neighbor.  


   Annotate Valley
Annotate the lowest valley in an area.  When this button is clicked, you can drag and create a selection box.  When the mouse button is released, the lowest valley in the selection box is annotated.  The definition of a valley is a point where it is lower than both its neighbors.  If a point is at one of the ends of a chromosome, it only needs to be lower than its only neighbor.


   Move Annotation
Press once to set the graph to move annotation mode.  Press again to get out of the mode.  While in this mode, you can click and drag annotations on the graph and move then wherever you want to.


   Configure Plot
Clicking on this button will cause a dialog box to appear that will let you change the amount of space between the graph and the window.  Will cause values to be set to default values at first, which is annoying.  May not be too useful, but there if you need it.

File Menu Bar

    


Open GRX File - Opens a .grx file.  This should not be used when you want to load data from a text file.  Use 'Import Data from File' instead.


Save GRX File - Save the current state of the graph as a .grrx file.  The state of the graph (chromosome, annotations, viewing limits, etc.) will be saved to a file and can be opened later on using 'Open GRX File'.


Save As GRX File - Same as 'Save GRX File', except you can choose a new filename.


Import Data from File - Load data to be graphed from a text file.  First a file open dialog box will appear asking you which file you want to load.  After the file is selected, the file format dialog box will appear.  Here is a screenshot of the dialog (Linux version).



The top of the dialog shows a preview of the text file.  The first five lines of the file are shown to help determine which columns to choose.  Note that comments can be used inside a text file.  Comments are denoted with a '#' at the beginning of the line, and will be ignored when processing the file.

The next line asks which column indicates position and which indicates ppl values.  Choose the correct columns from the drop down list.  The default for the position column is the second column. The default for the PPL column is the third column. The third option specifies how many lines make up the header of the file. For many files, there is a header at the top that has information about the file, like what type of information the columns stand for. The default for the header is one line.

The next part of the dialog asks whether this file has data for a single chromosome or multiple chromosomes.  The default is to use multiple chromosomes.  If single chromosome is chosen, you must also specify what number that chromosome is.  If multiple chromosomes are chosen, you have the option of specifying the column that indicates what number chromosome this line of data is associated with.  If 'None' is chosen for this value, then it is assumed that the data listed is for chromosome one, then chromosome two, etc.  Data for a chromosome ends and a new chromosome is started when the position value for the current line is less then the position value from the previous line.  The default is to have the first column be the column that specifies the chromosome number. If multiple chromosomes are selected, you can also specify to import all chromosome found in the file, or only import specific ones.  If only specific chromosomes are chosen, separate different chromosome numbers with a comma (e.g. '1, 4, 8').

The next text box will ask what the name of this line set is. A line set is defined as all the lines imported from one text file. For example, if a set of 23 chromosomes are imported from one file, that will be counted as the first line set. Then later on 23 more chromosomes are imported from another file (using a separate overlap option, for example), and those 23 are counted as a second set. The name of the set can be shown by the legend instead of the name of each individual line (see 'Options Menu Bar', 'Configure Graph'). Also the name of the set is used to identify the line set when trying to change the line properties of the sets (see 'Options Menu Bar', 'Configure Line Sets'). This box can be left blank.

The next two text boxes ask whether to attach a prefix or postfix to the chromosome numbers.  For example, if a postfix of 'a' is specified, then the chromosome number will be listed as 1a, 2a, etc.  Handy for when you overlap different chromosomes together, and you want their names to be different.

The next text box asks if there are any columns with extra data.  Any extra data for that point will be displayed at the lower left corner next to the position and ppl value of the mouse position.  For example, if each position has an associated marker to it listed in the file, that column can be listed.  Then when moving the pointer over the graph, the lower left corner will show the marker data of the closest point to the current mouse position.

The last line asks whether to keep imported lines the same color.  The default is to have it checked.  The color button next to it lets you choose what one color to use for all imported chromosomes.  If this is unchecked, each chromosome being imported will each have a different color, and the color box has no effect.

Note: Importing Data from File when there are already some chromosomes loaded in the graph will result in the chromosomes being placed at the end of the graph.  If you want to overlap different chromosomes together, see 'Overlap Menu Bar'.


Import Data from Remote File - Load a data file from a url.  When this is chosen, a dialog box will appear asking for the url of the remote file.  Simply type the entire path to the file (including http://), then the same file format dialog will appear as shown earlier.


Export Graph to Color Image - Save the current view as an image file.  This is equivalent to clicking on the Save button on the toolbar.  The preferred image format is png.


Export Graph to Grayscale Image - Same as 'Export Graph to Color Image', except that the final image will be grayscale.


Page Setup - Page Setup options for printing.  Currently lets you set the width of the figure and the margin.  There is no option to choose between landscape or portrait orientation.  The page orientation is automatically set using the width of the figure.  If the width is too large to fit using portrait, then landscape is used.


Print - Prints the current graph.

Edit Menu Bar




Clear Graph - Resets the graph and removes everything from it.


Copy Graph - Put the graph on the clipboard.


Remove Chromosome - Remove a chromosome from the graph.  A dialog box with all chromosomes in the graph listed will appear.  Simply check the ones you wish to remove, then click OK.

Options Menu Bar




Configure Graph - Lets you configure many aspects of how the graph is drawn.  The first three text boxes allow you to add a title, and labels to the x and y-axis. Then there are options that affect the entire graph, such as background color and the text size of the title and axis labels.  There are also options that allow you to show a legend for the graph, which is off by default.  Next to that is an option as to what the legend show. The legend can display each individual line's name, or it can show only the name of the different line sets. The next row includes options about the legend, specifically whether to show a frame around the legend, and where to place the legend. The next two rows of options deal with annotations. The first row has options to set the precision of the ppl value in the annotation text. If set to default, two decimal places are shown, except when the ppl value is less than 0.02, in which case three decimal places are show. If that is not desired, then a custom precision can be set. The next line has options to set the precision in regards to the position text in annotations. The next option lets you display annotations the same color as the line associated with it. Normally annotations are all black.  The option on the next line lets you suppress duplicate line numbers in the labels below the graph.  For example, if the label for a group of lines is normally '1, 1, 1', then choosing this option makes the label simply '1'. This option is helpful when overlapping many lines that have the same chromosome  number.

The bottom of the dialog has a list of all chromosome in the graph, with options to change attributes of a chromosome and the way they are drawn. See below for a screenshot of the dialog box (Linux version).

Note: for the title and the axis text, you have the ability to use mathematical expressions in them.  See 'Mathematical Expressions' for more info.




Configure Line Sets - Sets properties of lines sets.  The options that can be changed include the set name, color, line style, thickness, symbol, and symbol size. When the legend is set to show the name of the line sets, the name shown is the one in the first column. A screenshot of the dialog box is shown below (Linux version).




Configure All Lines - Sets properties of all lines at once. This is useful if you want to set the color, line style, or other properties of lines all at once without having to change each line manually.




Configure Gene Markers - Lets you change certain properties of gene markers (See Insert Menu Bar -> Gene Markers for more information).  A screenshot of the dialog box is below (Linux version). Each row represents one set of gene markers. The properties availabe to be configured are the gene marker's name, size, color, and symbol.




Set x-axis, y-axis limits - Lets you exactly determine the viewing limits of the x-axis and y-axis. A screenshot of the dialog box is below (Linux version). There are four textboxes, one for xmin, xmax, ymin, and ymax. For xmin and xmin, the values in the textbox are relative to a chromosome in the graph. Next to the xmin and xmax textboxes are choice boxes that let you select which chromosome to use as a reference point. The range column displays the current range of positions for the currently selected chromsome. You cannot use an x value that is outside the range of the chromosome. For y-axis values, simply type in the number you wish the view limits to be.


Insert Menu Bar



Insert Horizontal Line - Insert a horizontal line that goes through the entire graph.  See the toolbar button 'Insert horizontal line' for more info.


Insert Gene Markers - Insert a set of gene markers. A screenshot of the dialog box is shown below (Linux version). Gene markers are specific points on the chromosome that are marked by a certain symbol. The default is to use a circle as the symbol. When this option is chosen, a file dialog box will appear, asking for the text file to load. The file containing gene markers must have a least two columns. One column must indicate the chromosome number, and the other column must indicate the position along the chromosome of the gene marker. The dialog box also allows you to give the set of gene markers a name, and change their size, color and symbol.  The gene marker name is only useful in the case when you load multiple sets of gene markers and later configure them. The name will be displayed so that you can differentiate between them.



Overlap Menu Bar



Insert Overlapping Chromosome from File - Import chromosome(s) from a text file, and overlap these new chromosomes with existing chromosomes in the graph.  When you first click this option, a file open dialog will appear, and you choose what file to import the data from.  Once the file is chosen, a file format dialog appears that is almost identical to the one shown earlier.  The only difference is that at the bottom there is a drop list with the label 'Chromosome to begin overlap with'.  Here you choose which existing chromosome where the overlapping will begin.  For example, if the graph is currently loaded with chromosome 3, 4, 7, 12 and you import chromosomes 9, 14, 20 from a file and choose chromosome 4 as the chromosome to begin overlapping with, then chromosome 9 will overlap with 4, 14 will overlap with 7, and 20 will overlap with 12.


Overlap existing chromosomes - Takes an existing chromosome(s) from the graph and moves it so it overlaps with another existing chromosome.  When you choose this option, a dialog box like the one below will appear.  At the top is a list of all chromosomes currently in the graph.  You can select as many as you want to move.  At the bottom is a drop down box that lets you select where to move all the previously selected chromosomes.

Annotations



Annotations are used to highlight and label points of interest.



Add Annotation
To add an annotation, use one of the annotation buttons on the toolbar.  See 'Toolbar Buttons' for more info.



Remove Annotation
To remove an annotation, simply right-click on the annotation, and select Remove Annotation.




Editing Annotations
The default text for an annotation is to display the ppl value as a percent, then the '@' symbol, then the position of that point in centimorgans.  To edit the text of an annotation, simple right-click on the annotation and select Edit Annotation.  Note that you can use mathematical expressions for annotation text.  See 'Mathematical Expressions'.


Annotation Layout
There is an automatic layout algorithm for the annotations.  It tries to place annotations in such a way that it will not overlap with other annotations or lines in the graph.  The algorithm is not perfect, and will occasionally break, especially when there are many annotations in a neighborhood.  The best remedy for fixing this is to manually move the annotations yourself.  (See 'Move Annotation' in Toolbar Buttons).  Once you move an annotation, you are forever responsible for it. The automatic layout algorithm will not move it anymore. This applies as long as you are in the same view. You go to a different view, for example, when you zoom in. If you pan, the annotations will stay in place.  They will also remember positions between views when using the back and forward buttons to change the view.  The automatic layout algorithm will initiate when a new view is created.

Command Line Options



There are command line options available to perform most actions in graphApp.
The syntax is to type the following at the command line:

graphApp <command1> <args1> <command2> <args2> etc...

If a command appears multiple times, only the first occurance is used, unless
the command works on only one chr, in which case it can be  specified multiple
times. Other commands that can be used multiple times are specified in their
description.

Commands are performed in the order that they are specified, except in some
special cases:
    -open is always performed first.
    -save and -export are always performed last.

Command arguments that are column indices start at 1, not 0.

Some command line arguments ask to specify a color. There are three options
available to specify color:
    1. Some colors are identified by a single letter. All available colors like
       this are:
            'b' - blue, 'g' - green, 'r' - red, 'c' - cyan, 'm' - magenta,
            'y' - yellow, 'k' - black, 'w' - white
    2. You can specify colors by a hex value, for example '#0022ff'
       (Note the '#' instead of 'x' for a hex value)
    3. You can specify a name of a color in the X11 color set, for example
       'Indigo', 'PapayaWhip', or 'Olive'. For a listing of what colors are
       available, click here, or goto
       http://www.mcfedries.com/books/cightml/x11color.htm.
       Note that all colors may not work.


-open <type> <filename>
    * <type> --> 'text', 'graph', 'grx', what type of file to open
    * <filename> --> path to file

    * all these commands are used when <type> is 'text'
    * command are optional unless otherwise specified
   
    -pos_col <num>
        * column index of column with position values
        * <num> --> integer, the column index
        * required

    -ppl_col <num>
        * column index of column with ppl values
        * <num> --> integer, the column index
        * required
   
-header_lines <num>
        * the number of lines at the top of the file that constitute the header.
        * the header will be ignored when processing the file.
        * <num> --> integer, the number of lines

    -chr_num <num>
        * if text file has info about only one chr, tells program what the
          chr number of that chr is
        * <num> --> integer, the number of the chr in the file

    -chr_col <num>
        * column index of column with chr number
        * <num> --> integer, the column index
        * optional, if left out, will assume first set of values is chr1,
          then second set of values is chr2, etc.

    -chr_select <num1> <num2> etc.
        * select only some chr's to be loaded in
        * <num1> --> integer, the number of the chr that should be loaded

    -chr_prefix <prefix>
        * changes name of chr to add a prefix string before every chr number
        * <prefix> --> string

    -chr_postfix <postfix>
        * changes name of chr to add a postfix string after every chr number
        * <postfix> --> string

    -extra_data_col <num1> <num2> etc.
        * designate columns that contain extra data associated with each point
        * used to specify a marker # for each point, then when moving the
          cursor, the graph would display what marker # the closest point to
          the mouse curser is at
        * <num1> --> integer, the column index

    -chr_same_color <bool>
        * tells program to either keep all chr's the same color,
          or to make each chr a different color
        * <bool> --> 'on', 'off'
        * optional, if not present, value is 'on'

    -chr_color_all <color>
        * set the same color for all chrs
        * only works if -chr_same_color is 'on'
        * <color> --> color

    -lineset_name <name>
        * gives a name to this line set
        * <name> --> string, the name of the set

-save <filename>
    * save a graph as a .grx file
    * <filename> --> path to a file
    * if .grx is not at end of filename, it is added on automatically
    * can be specified multiple times

-save_graph <filename>
    * save a graph as a .graph file
    * <filename> --> path to a file
    * if .graph is not at end of filename, it is added on automatically
    * can be specified multiple times

-export <filename>
    * export graph to an image
    * <filename> --> path to a file
    * file format is determined by file extension in filename. only .eps
      and .png files work. if no file extension, or an unrecognized one,
      .png will be added to filename
    * can be specified multiple times

-noshow
    * does not show the window
    * closes the program as soon as the other commands are done

-comment <comment>
    * specify the comments on the side of the graph
    * <comment> --> a string (be sure to put quotes around the comment)

-hline <num1> <num2> etc.
    * specify horizontal line(s) in graph
    * <num1> --> number, the y-value (ppl-value) of the line(s)
    * can be specified multiple times
   
-title <label>
    * set the title of the graph, will appear above graph
    * <label> --> string

-xlabel <label>
    * set the label of the x-axis
    * <label> --> string

-ylabel <label>
    * set the label of the y-axis
    * <label> --> string

-bgcolor <color>
    * set background color
    * <color> --> color

-title_text_size <num>
    * the font size of the graph title
    * <num> --> integer
    * default is 14

-axis_text_size <num>
    * the font size of the x and y-axis text
    * <num> --> integer
    * default is 12

-show_legend
    * show the legend
    * default is to not show legend

-show_legend_frame <bool>
    * show a frame around the legend
    * <bool> --> 'on' or 'off'
    * default is 'on'

-legend_show_linesets <bool>
    * <bool> --> 'on' or 'off'
    * if 'on', the legend lists the line sets
    * if 'off', the legend lists each individual line
    * default is 'off'

-legend_location <loc>
    * the location of the legend
    * <loc> --> 'upper right' , 'upper left', 'upper center', 'lower right', 
                'lower left', 'lower center', 'center left',  'center right',
                'right', 'center'
    * default is 'upper right'

-ann_ppl_precision <arg1>
    * specifies the amount of ppl precision to display in annotations
    * <arg1> --> 'default' or integer
    * with 'default', ppl is shown with two decimal places (e.g. 7%), except
      when the ppl is less than .02, in which can it is shown to three
      decimal places (e.g. 7.2%)

-ann_pos_precision <num>
    * specifies how many decimal places to show for the position number
      in the text of an annotation
    * <num> --> integer, how many decimal places to show
    * default is 0

-ann_same_color
    * specifies that annotations should be the same color as the line it is
      associated with. only works with annotations specifying peaks and valleys
    * default is that all annotations are black

-no_duplicate_numbers
    * when chromosome numbers overlap and they are the same number,
      do not show the number multiple times in the labels underneath
      the graph. for example, '1, 1, 1' becomes '1'

-lineset_change_name <oldname> <newname>
    * change the name of a line set, changes the name in the legend if the
      legend is set to display line set names
    * <oldname> --> the old name of the line set
    * <newname> --> the new name of the line set

-lineset_color <name> <color>
    * change the color of all chrs in a line set
    * <name> --> label, the name of the line set
    * <color> --> color

-lineset_linestyle <name> <style>
    * set the line style of all chrs in a line set
    * <name> --> label, the name of the line set
    * <style> --> the style of the line
    * valid styles are: 'solid', 'dashed', 'dotted', 'dash-dot',
                        'steps', 'no line'
    * default is 'solid'

-lineset_thickness <name> <thickness>
    * change the thickness of all chrs in a line set
    * <name> --> label, the name of the line set
    * <thickness> --> integer, how thick the chr lines should be
    * default is 2

-lineset_symbol <name> <symbol>
    * change the symbol that represents each data point
    * <name> --> label, the name of the line set
    * <symbol> --> symbol
    * valid symbols are: 'no symbol', 'square', '+', 'o', 'x', 'diamond',
                         'thin diamond', '^', 'v', '>', '<', '|', '_',
                         'hexagon', 'pentagon', 'tripod 1', 'tripod 2'
    * default is 'no symbol'

-lineset_symbol_size <name> <size>
    * change the size of the symbol for all chrs in a line set
    * <name> --> label, the name of the line set
    * <size> --> integer, new size of the symbol
    * default is 6

-chr_name <oldname> <newname>
    * change the name of a chr, changes what is displayed at the
      bottom of the graph
    * <oldname> --> the old name of the chr, when chr's are first loaded,
                    their name is simply their chr number, unless a prefix or
                    postfix was applied
    * <newname> --> the new name of the chr

-chr_color <name> <color>
    * change the color of a chr
    * <name> --> label, the name of the chr
    * <color> --> color

-chr_linestyle <name> <style>
    * set the line style of a chr
    * <name> --> label, the name of the chr
    * <style> --> the style of the line
    * valid styles are: 'solid', 'dashed', 'dotted', 'dash-dot',
                        'steps', 'no line'
    * default is 'solid'

-chr_thickness <name> <thickness>
    * change the thickness of a chr
    * <name> --> label, the name of the chr
    * <thickness> --> integer, how thick the chr line should be
    * default is 2

-chr_symbol <name> <symbol>
    * change the symbol that represents each data point
    * <name> --> label, the name of the chr
    * <symbol> --> symbol
    * valid symbols are: 'no symbol', 'square', '+', 'o', 'x', 'diamond',
                         'thin diamond', '^', 'v', '>', '<', '|', '_',
